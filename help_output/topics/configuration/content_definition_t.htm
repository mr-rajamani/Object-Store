Create Content Definition

Create the content definition in the local packs.

The content definition captures the definition of the domain object that you want to model.

The Type of definition created for the local packs are:
  • Content - captures the definition of the domain object that you want to model.
  • Property - is a data type in programming parlance. When you share the content pack with a user, then the user can use the authored properties as well.
  • Global Action - Actions defined globally.
  • Global Event - Events defined globally.

Perform the following steps to create a content definition.

  1. Click next to the created local pack.
  2. Select Content from the Type of definition drop down list.
    The Create Content Definition is displayed.
  3. Enter the Content definition name.
    Note: The content definition name should start with an Uppercase character.
  4. Select the Extends from content definition from the drop-down list.
    By default, only the system content packs are displayed for an empty database. It will display user deployed content packs if they are available.
  5. Enter the Description.
    The description is mandatory in all the places for documentation purpose. The information provided in this field will generate documentation for the content definition.
  6. Optional: Select Abstract content definition to define an interface. The content definition created will be extended and defined later.
  7. Click Create.
    The local packs lists the created content definition.
  8. Click the created content definition to view the Content Outline in the bottom pane.
    The content outline of the content definition will be displayed.

    The content outline provides you to further author the content definitions like Properties, Relations, Actions, and Events.
