    Object Store is an enhanced next-generation platform core that has combined capabilities of numerous services.

  • Object Store Features

    Altair Object Store is an advanced content management system that combines the capabilities of numerous services and an action execution engine that applies Access Control List (ACL) and other authorization policies for enhanced security.

  • Benefits of Object Store

    The Altair Object Store allows you to model engineering objects that are independent of the user interface and can be consumed in a variety of applications via APIs.

  • Altair Object Store Platform Architecture

    The Object Store architecture includes, in addition to the central Object Store engine that manages configuration and execution, an environment manager, a multi-model database, an event bus, and gateways. Investigate the many different parts that make up the Object Store architecture.

Create Property for the Content Definition

Author the content definition by creating a property.

For our ToDo content definition let's add a description property which will take string values.

  1. Click next to the Property.
    The Create Property is displayed.
  2. Enter the Property name.
  3. Select the Property Type from the drop-down list.
    The property type is a list of various native types supported by JSON schema and the system properties authored on top of the JSON schema.
    Note: Additional fields are displayed based on the selected Property Type.
    For example, if you have selected String as the Property Type, then it displays the Enumerations for property value field.
  4. Enter the Description.
    The description is mandatory in all the places for documentation purpose. The information provided in this field will generate documentation for the content definition property.
  5. Optional: Select Required to define if property defined is required or not for the content definition.
  6. Click Create.
    The content definition lists the created property.
In this example, we have created a description property to describe the task. Now, we can create status and category properties to describe the task status and its category.
